1. 概要
	立花証券ｅ支店ＡＰＩ（以下 e_api）を python から利用するためのサンプルプログラムです。
	以下機能の動作確認がでます。

	(1) 認証I/F 認証実施、通番及び仮想ＵＲＬをファイル保存及び読出機能
	(2) REQUEST I/F 指定銘柄、指定項目の時価問い合わせ取得機能
	(3) EVENT I/F 指定銘柄の時価配信取得機能
	(4) EVENT I/F WebSocket 指定銘柄の時価配信取得機能


2. 使い方
	エディター等で e_api_sample.py を開き以下を変更。

	(1) 認証URL、ログインＩＤ、パスワード
		「
			gp_url_auth	= "https://demo-kabuka.e-shiten.jp/e_api_v4r7/auth/"
			gp_usid		= "username"
			gp_pswd		= "password"
		」
		認証URLはデフォルトデモ環境です。
		本番環境やバージョンアップ時など必要に応じ書き換えてご利用ください。

	(2) 機能選択
		「
			gp_proc_request_if	= 0	#! use request i/f sample get market price.
			gp_proc_event_if	= 0	#! use event i/f specify issue_code's and columns.
			gp_proc_event_ws	= 0	#! use event i/f websocket.
		」
		1.(2) 利用時は gp_proc_request_if = 1 に変更。
		1.(3) 利用時は gp_proc_event_if	= 1 に変更。
		1.(4) 利用時は gp_proc_event_ws	= 1 に変更。
		※(3),(4) は  Ctrl^C 押下まで受信ループします。


	(3) HTTPS POST または HTTPS GET の利用選択
		「
			self._p_debug_http_post	= 0	#! request to http-post or http-get.
		」
		上記設定で (1), (2) について 1:POST または 0:GET の切替が可能。

	(4) その他
		その他デバック出力用のフラグ「self._p_debug_print_oxox」は
		必要に応じ ON/OFF を切り替えてご利用ください。


3. 処理内容
	（初回等、保存ファイルが無い場合）ログイン認証を実行し通番と仮想URLをファイルに保存。
	その後、1.(2)-(4)について指定処理を実施。
	1.(2) の場合、時価問合取得要求送信後、応答内容（JSON）を画面に出力し
	通番（p_no）を再度保存ファイルに保存。

	保存ファイル名は YYYYMMDD_e_api_sampl.txt です。

	保存ファイルがある場合は該当ファイルから通番、仮想URLを取得し認証をしません。
	仮想URL の無効化などにより再度認証したい場合は
	保存ファイルを削除してから起動してください。

	保存ファイルが無い場合は認証し仮想URLを取得、通番は初期化します。
	保存ファイル名には日付（YYYYMMDD）を付けています。
	翌日になるとファイル名が変わり読み込まれないため認証します。


4. 実行方法
	% python e_api_sample.py


5. 実行環境
	python、及び import 利用の各ライブラリモジュールをインストールする。
	例：
		% pip install urllib3
		% pip install requests
		% pip install asyncio
		% pip install websockets
		など。

	windows の場合は cmd.exe で
	% python
	と入力しリターン押下すると python のインストーラが起します。
	起動したインストーラ画面に従いインストールを実施下さい。
	※インストーラが起動しない場合、
	　python のパッケージをダウンロードするなどしインストールして下さい。

	その後、cmd.exe で上記ライブラリをインストールして下さい。
	※動作環境構築については自己責任で実施願います。


6. 動作確認環境
	以下環境で動作確認済です。
	(1) Python 3.13.5 on CentOS Linux release 7.3.1611 (Core)
	(2) Python 3.8.12 on FreeBSD 13.0-RELEASE-p14 (GENERIC)
	(3) Python 3.13.5 on Window 11 Pro 24H2
	※利用ＯＳやインストールされている各種アプリケーションを含むソフトウエア、
　　　　　python（含む import ライブラリ）等のバージョンやそれらの組み合わせなどにより
　　　　　動作しないケースもありますので、予めご了承願います。


7. 実行エラー
	(1) 稼働時間外
		「
			error, e_api request error.
			p_errno:        [-62]
			p_err  :        [システム、情報提供時間外。]
		」

	(2) ログインエラー
		「
			error, e_api application error.
			sResultCode:    [10031]
			sResultText:    [ログイン前に行う電話番号認証が認証されない、または、ユーザID、暗証番号をお間違えです。]
		」

	(3) セッション無効化時
		「
			error, e_api request error.
			p_errno:        [2]
			p_err:		[セッションが切断しました。]
		」
