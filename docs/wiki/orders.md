# 注文（立花証券 e支店）

立花証券 e支店 API を使った注文発注・照会・余力確認の操作ガイド。

> 注文機能は立花証券のセッションが必要。接続方法は [設定・カスタマイズ](settings.md#立花証券-の接続) を参照。

---

## 注文パネルを開く

1. 左サイドバーの **🖊（鉛筆）** ボタンをクリック
2. インラインメニューが展開する:
   ```
   [ Order Entry  ]  ← 注文入力
   [ Order List   ]  ← 注文照会
   [ Buying Power ]  ← 余力情報
   ```
3. いずれかを選ぶと、フォーカス中のペインが Horizontal Split され、新ペインにパネルが開く

> 鉛筆ボタンと検索ボタン（🔍）は相互排他。片方を開くと他方が閉じる。

---

## 注文入力パネル（Order Entry）

```
┌─────────────────────────────────┐
│  [買い]  [売り]                  │
├─────────────────────────────────┤
│  銘柄: 7203 トヨタ自動車         │  ← チャートペインと連動して自動更新
│─────────────────────────────────│
│  口座: [特定 ▼]                  │
│  数量: [____100____] [全数量]    │  ← [全数量]: 売り時のみ表示
│         (保有: 200株)            │  ← 売り時のみ表示
│  価格: [成行 ▼] [▼][________][▲]│  ← ▲▼: 呼値単位で±1
│  期日: [当日 ▼]                  │
│─────────────────────────────────│
│  受渡金額: ¥XXX,XXX (概算)       │
│  手数料:   ¥YYY                  │
│─────────────────────────────────│
│  発注パスワード: [__________]    │
│─────────────────────────────────│
│       [  注文確認  ]             │
└─────────────────────────────────┘
```

### 各項目の説明

| 項目 | 説明 |
|------|------|
| **売買区分** | 買い / 売り |
| **銘柄** | チャートペインの銘柄に自動連動。手動変更不可 |
| **口座** | 特定 / 一般 / NISA / N成長 |
| **数量** | 注文株数。売り時は [全数量] ボタンで保有株数をセット |
| **価格** | 成行（`*`）または指値（数値）。▲▼ ボタンで呼値単位ステップ |
| **期日** | 当日 / 日付指定（10営業日以内） |
| **発注パスワード** | 立花証券の第二パスワード（マスク表示。ログ・ファイルには記録されない） |

### 発注手順

1. 売買区分・口座・数量・価格・期日を設定
2. 発注パスワードを入力
3. **「注文確認」** ボタンをクリック
4. 確認モーダルで内容を確認:
   ```
   銘柄:  7203 トヨタ自動車
   売買:  買い
   数量:  100株
   価格:  成行
   口座:  特定
   [キャンセル]  [注文を発注する]
   ```
5. **「注文を発注する」** で発注完了

### REPLAY モード中の動作

REPLAY モード中は仮想注文モードになる:

- パスワード入力欄が非表示
- バナーに **「⏪ REPLAYモード中 — 注文は無効です」** が表示
- ボタンラベルが **「仮想注文確認」** に変わる
- 発注 → 証券 API を呼ばず `VirtualExchangeEngine` で処理

詳細は [リプレイ / 仮想取引](replay.md#仮想取引replay-モード中) を参照。

## 注文照会パネル（Order List）

当日の注文一覧を表示する。

```
銘柄    売買  注文株数  約定株数  注文単価  約定単価  状態      注文日時
7203    買   100      100      成行     3,250   全部約定  09:30:22
8411    売    50        0      3,100      —     注文中    10:15:08
...

                                                        [更新]
```

### 状態の色分け

| 状態 | 色 |
|------|-----|
| 全部約定 | 通常色 |
| 一部約定 | 黄色（強調） |
| 受付中 / 注文中 | 薄色 |
| 取消完了 | グレー |
| エラー系 | 赤 |

### 行の操作

**クリック** → 約定明細を展開:
```
  約定明細
  09:30:22   3,250   100株
```

**[訂正] ボタン**（注文中・受付中の行のみ表示）→ 訂正モーダル:
```
新しい価格: [________]
新しい数量: [________]
パスワード: [________]
[キャンセル]  [訂正する]
```

**[取消] ボタン**（注文中・受付中・一部約定の行のみ表示）→ 取消確認モーダル:
```
注文番号 XXXXXXXX を取消しますか？
パスワード: [________]
[キャンセル]  [取消する]
```

### 更新

- **[更新] ボタン**: 手動で最新状態を取得
- **約定通知**: 「全部約定」に状態遷移した注文は Toast 通知が出る
- 自動ポーリング: 現在未実装（手動更新のみ）

## 余力情報パネル（Buying Power）

```
現物口座:
  現物株買付可能額:    ¥3,250,000
  NISA成長投資残高:    ¥1,000,000

信用口座:
  信用新規建可能額:    ¥8,500,000
  委託保証金率:        35.50%
  追証:               警告なし

                              [更新]
```

**追証確定時**: 「⚠ 追証確定」と赤色で表示される。

### 更新

- パネルを開いた時: 自動取得
- **[更新] ボタン**: 手動で最新状態を取得
- 注文発注後の自動更新: 現在未実装

## 信用取引の種別

| 表示 | 意味 |
|------|------|
| 現物 | 現物取引 |
| 信用新規（制度 6 ヶ月） | 制度信用 新規建て（貸株期間 6 ヶ月） |
| 信用返済（制度） | 制度信用 返済 |
| 信用新規（一般） | 一般信用 新規建て |
| 信用返済（一般） | 一般信用 返済 |

## 制約・注意事項

- **板情報（DOM）は東証立会時間のみ更新**（9:00-11:30, 12:30-15:30 JST）。立会時間外はデータが止まる
- **日足チャートのみ**。分足・時間足は立花証券 API が非提供
- **逆指値注文は未実装**（将来対応予定）
- **自動ポーリング（注文一覧・余力）は未実装**。[更新] ボタンで手動取得
- **注文発注後の余力・注文一覧の自動更新は未実装**
- 第二パスワードはメモリにのみ保持。ログ・設定ファイルには書き込まれない
